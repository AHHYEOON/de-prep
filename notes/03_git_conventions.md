# Git Commit Convention

## 타입 목록
- feat: 새 기능
- fix: 버그 수정
- docs: 문서 (README, 주석)
- style: 포맷 (의미 변경 없음)
- refactor: 구조 개선 (기능 유지)
- test: 테스트
- chore: 설정/잡일
- perf: 성능 개선

## 학습용 추가
- solve: 코딩 문제 풀이

## 규칙
- 소문자, 콜론 뒤 공백
- 50자 이내
- 명령형/현재형

## 커밋 메시지 대소문자 규칙 (내 스타일)
- 타입은 소문자 고정 (solve, docs, feat...)
- 설명 시작은 소문자 선호 (day 1, docs 추가...)
- 고유명사/기술명은 원래 표기
  - LeetCode, Python, SQL, Two Sum, PostgreSQL 등
- 일관성 유지가 제일 중요
